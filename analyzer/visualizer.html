Welcome to Kestrel

What would you like to do?

🔬
First time using Kestrel?

Get started by adding your photo folders to the Analysis Queue. Kestrel will automatically detect birds, identify species, and score image quality across your entire library.

Analyze Photos…
📂
Returning to review your photos?

Open a folder you’ve already analyzed with Kestrel. Use the search and filtering options on the left to find specific species or sort by image quality.

Open Folder…
💡 Load multiple folders at once! Open a parent folder and explore your entire library using the Folder Tree on the left.
💡 Hold Ctrl and click scenes to select multiple — then merge them into a single scene with one click.
💡 Found a photo you love? Double-click any scene to view it, rate photos with ⭐ stars, and hit Save Changes to keep your ratings. Use the Only show manually-rated scenes filter to find them again later.
💡 Want to open photos in your preferred editor? Double-click any photo in a scene — or configure your editor via Settings…
Need help? Visit projectkestrel.org/tutorial for a video walk-through or look for the in-app Tutorial buttons.

How many subfolder levels to scan when building the folder tree (1–6). Higher values may increase scan time.
Controls how quality scores are converted to star ratings. Per folder ensures each folder has a spread of 1–5 stars regardless of overall quality. Global ranks photos against all folders you have analyzed. None uses fixed quality thresholds.
Minimum confidence score (0.10–0.99) that the bird detector must reach before Kestrel counts something as a detection. Lowering this value may increase false positive detections on difficult images. Requires re-analysis to take effect. Also filters which species and family tags are shown in scene dialogs.
Images captured within this many seconds of each other are treated as the same burst/scene (using EXIF timestamp). Requires re-analysis to take effect.
Confidence threshold for mask segmentation (0.50–0.95). Higher values produce tighter masks around birds, lower values are more inclusive. Requires re-analysis to take effect.

📊 Help Improve Project Kestrel

Would you like to share anonymous usage data to help improve the app? This is completely optional and you can change your mind at any time.

What would be shared (anonymously):

  • Number and size of images analyzed
  • File formats used (e.g. CR3, JPG)
  • Analysis duration
  • App version and OS type

What is never shared:

  • Filenames, folder paths, or image content
  • Any personally identifiable information

Your machine is identified only by a random ID — no device name, username, or IP address is stored or sent.
